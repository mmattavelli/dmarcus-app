DMARC Policy Guide

Specifies the policy to apply to emails that fail DMARC checks:

  • none: Take no action (monitoring only)
  • quarantine: Treat as suspicious (e.g., send to spam)
  • reject: Reject the message entirely

Recommended: Start with none, then move to quarantine, and finally reject after monitoring results.

Specifies the percentage of messages to which the DMARC policy is to be applied.

Useful for gradually rolling out a new policy:

  • Start with 100% when in none mode
  • When changing to quarantine or reject, start with a lower percentage (e.g., 10%)
  • Gradually increase to 100% as you verify legitimate emails aren't being affected

rua: Addresses to which aggregate reports should be sent (recommended)

ruf: Addresses to which forensic (failure) reports should be sent (can generate high volume)

Format: mailto:reports@example.com

Multiple addresses can be specified, separated by commas.
